Effects (data-effect)

Selective reactivity with @use

Without @use, an effect reruns whenever any $variable it reads changes. @use(['key']) restricts reruns to listed keys only.

Computed / derived values

Derive values from multiple store keys in a single effect.

Class, style, hidden

Any DOM property can be driven reactively by a data-effect expression.

el, store, and $() in expressions

el is the current DOM node. store is the raw proxy (no $ prefix). $(parent, selector) is querySelector.

@set — array element mutation with re-render

Direct assignment like $colors[0] = 'x' bypasses the Proxy and won't trigger a re-render. Use @set('key', index, value) instead.

Network — Links, Forms & Fetch

Link interception

Same-origin link clicks are intercepted and fetched via ssr.js (triggering a full-page swap). Use target="_top" to opt out.

  • index.html — intercepted, fetched via ssr.js
  • index.html with target="_top" — browser navigates normally
  • https://thorsenlabs.com — external, not intercepted

data-history on links

Controls whether history.pushState, replaceState, or nothing is called.

  • data-history="pushState" (default) — adds to history
  • data-history="replaceState" — replaces current entry
  • data-history="none" — fetch but no URL change

Form — GET / POST

Submitting this form POSTs and swaps the response.

@get / @post — programmatic fetch

@get(url) and @post(url, options) trigger a fetch from any event handler. HTML responses update the DOM via ssr:sse-patch-elements.

ssr:sse-patch-elements — non-HTML / non-SSE responses

When a fetch returns a content type other than text/html or text/event-stream, ssr:sse-patch-elements fires so you can handle it yourself — e.g. parse application/json.

Trigger a @get or @post to an endpoint that returns JSON to see this event fire.

ssr-headers — default request headers

A <meta name="ssr-headers"> in <head> sends extra headers on every ssr.js fetch. Useful for CSRF tokens.

<!-- place in <head> -->
<meta name="ssr-headers" content="'x-csrf-token': 'abc123', 'x-requested-with': 'XMLHttpRequest'">
